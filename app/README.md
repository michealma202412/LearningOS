## 🚀 环境准备与启动（重要！首次使用必读）

### 🔴 常见问题：点击"开始录音"没反应

**原因**：浏览器的 MediaRecorder API 和 getUserMedia 有严格的安全限制。

**解决方案**：必须通过 localhost 或 HTTPS 访问，不能直接双击 HTML 文件（file:// 协议）。

---

### ✅ 正确启动步骤

#### 1. 安装 Node.js（首次使用必需）

**方法A：官方安装包（推荐）**
1. 访问：https://nodejs.org/
2. 下载 **LTS 版本**（长期支持版）
3. 安装完成后，**完全关闭并重新打开 Git Bash**

**方法B：Windows Winget**
```bash
winget install OpenJS.NodeJS.LTS
```

**验证安装**：
```bash
node --version
npm --version
```
如果显示版本号（如 `v20.x.x`），说明安装成功！

> 💡 **注意**：安装后必须重启终端才能生效

---

#### 2. 进入正确的目录

```bash
cd d:/001_temp/02_EduWeb
```

> ⚠️ **常见错误**：在错误的目录运行命令会导致 "package.json not found" 错误

---

#### 3. 启动开发服务器

**方法A：使用启动脚本（最简单，推荐）**
```bash
./start.bat
```

**方法B：手动启动**
```bash
npm run dev
```

成功后会看到：
```
VITE v5.x.x  ready in xxx ms

➜  Local:   http://localhost:5173/
```

---

#### 4. 访问应用

在浏览器中打开：
- 📝 **新版统一入口**：http://localhost:5173/app/index.html ⭐ 推荐使用

---

### 🐛 故障排除清单

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| `npm: command not found` | Node.js 未安装或未重启终端 | 安装 Node.js 并重启终端 |
| `package.json not found` | 目录错误 | `cd d:/001_temp/02_EduWeb` 进入项目根目录 |
| 录音按钮无反应 | 使用了 file:// 协议 | 必须通过 localhost:5173 访问 |
| 麦克风权限被拒绝 | 浏览器安全策略 | 点击地址栏图标允许权限 |
| Console 报错 "getUserMedia" | 环境不符合要求 | 确保使用 localhost 而非 file:// |

## 🎤 如何使用

### 打开应用

访问：http://localhost:5173/app/index.html

你会看到：
- 📝 记录区（默认显示）
- 📁 文件列表（可折叠）
- 🔄 今日复习（可折叠）
- 📱 底部导航栏

---

### 开始录音

1. 选择模式："🎤+📝 录音+文字"（推荐）
2. 点击 **"▶ 开始录音"** 按钮
3. 系统会请求麦克风权限（首次使用）
4. 授予权限后，开始说话
5. 文字会实时出现在下方文本框
6. 点击 **"⏹ 停止"** 结束录音
7. 可以编辑文字内容
8. 点击 **"💾 保存"**

**保存内容包括**:
- 音频 Blob 文件
- 文字内容
- 当前日期文件夹信息
- 复习计划（[0,1,3,7] 天）

---

### 查看文件

1. 点击底部导航的 **"📁 文件"** 或展开文件列表区
2. 看到按日期分组的文件夹
3. 使用搜索框过滤
4. 点击文件夹展开查看详情

---

### 每日复习

1. 点击底部导航的 **"🔄 复习"** 或展开复习区
2. 系统自动显示今日到期的复习项目
3. 点击 ▶ 播放音频
4. 点击 **"✓ 完成"** 标记已复习
5. 已完成的项目会从列表中消失

## 📁 文件夹系统

### 自动分组

所有录音会按**创建日期**自动分组：

```
📁 2026-04-20
  - 🎤 音频1 (10:30)
  - 📝 文章1 (11:00)
  
📁 2026-04-19
  - 🎤 音频2 (15:20)
```

### 复习显示逻辑

复习区**只显示**满足两个条件的音频：

1. **创建日期的复习计划到期了**（根据艾宾浩斯算法）
2. **尚未标记为完成** (done = false)

---

## 🧠 艾宾浩斯算法

每条新录音会自动生成 4 个复习副本：

| 复习次数 | 时间 | 说明 |
|---------|------|------|
| 第1次 | 今天 | 立即复习（巩固印象） |
| 第2次 | 1天后 | 强化记忆 |
| 第3次 | 3天后 | 加深理解 |
| 第4次 | 7天后 | 长期记忆 |

**示例**:
- 2026-04-20 录制的音频
- 会在以下日期出现在复习区：
  - ✓ 2026-04-20（当天）
  - ✓ 2026-04-21（1天后）
  - ✓ 2026-04-23（3天后）
  - ✓ 2026-04-27（7天后）

## 🐛 常见问题

### Q1: 为什么看不到之前录音的音频？

**可能原因**:
1. 该音频的复习计划还未到达今天
2. 该音频已被标记为完成

**解决方案**:
- 在"文件列表"中查看所有历史记录
- 等待复习计划到期（根据艾宾浩斯算法）
- 或创建新的录音进行复习

---

### Q2: 浏览器提示"需要麦克风权限"

**解决方案**:
1. 点击 "▶ 开始录音" 时，浏览器会弹出权限请求
2. 选择 **"允许"**
3. 重新点击 "▶ 开始录音" 即可

**如果还是不行**:
- 检查系统是否允许浏览器访问麦克风
- 尝试用无痕模式打开页面
- 清除浏览器缓存后重试

---

### Q3: 语音识别不准确怎么办？

**解决方案**:
1. 使用 Chrome 浏览器（最佳支持）
2. 确保网络连接稳定
3. 说话清晰，语速适中
4. 在安静环境中使用
5. 手动修正识别错误的文字

---

### Q4: 如何导出我的学习数据？

第一阶段不支持导出（计划在第三阶段实现）

**临时方案**:
- 使用浏览器开发者工具（F12）
- 访问 Application → IndexedDB → learning_os → items
- 可以查看所有存储的数据

## 🎓 了解你的数据

### 数据对象结构

了解数据如何存储（开发者参考）：

```javascript
{
  id: "1244570420000-audio",     // 唯一标识（时间戳-类型）
  type: "audio",                  // 类型：audio 或 article
  blob: Blob {size: 12345},      // 音频文件（仅音频）
  content: "文章内容...",         // 文字内容（仅文章）
  createdAt: 1244570420000,      // 创建时间戳
  folder: "2026-04-20",          // 所属日期文件夹
  reviews: [                      // 复习计划（仅音频）
    { date: 1244570420000, done: false },  // 今天复习
    { date: 1244656820000, done: false },  // 1天后
    { date: 1244829620000, done: true },   // 3天后（已完成✓）
    { date: 1245175220000, done: false }   // 7天后
  ]
}
```
