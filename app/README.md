## ⚙️ 环境准备与启动（重要！首次使用必读）

### 🔴 常见问题：点击"开始录音"没反应

**原因**：浏览器的 MediaRecorder API 和 getUserMedia 有严格的安全限制。

**解决方案**：必须通过 localhost 或 HTTPS 访问，不能直接双击 HTML 文件（file:// 协议）。

---

### ✅ 正确启动步骤

#### 1. 安装 Node.js（首次使用必需）

**方法A：官方安装包（推荐）**
1. 访问：https://nodejs.org/
2. 下载 **LTS 版本**（长期支持版）
3. 安装完成后，**完全关闭并重新打开 Git Bash**

**方法B：Windows Winget**
```bash
winget install OpenJS.NodeJS.LTS
```

**验证安装**：
```bash
node --version
npm --version
```
如果显示版本号（如 `v20.x.x`），说明安装成功！

> 💡 **注意**：安装后必须重启终端才能生效

---

#### 2. 进入正确的目录

```bash
cd /c/2_Workspace/WS/006_LearningOS/LearningOS
```

> ⚠️ **常见错误**：在错误的目录运行命令会导致 "package.json not found" 错误

---

#### 3. 启动开发服务器

**方法A：使用启动脚本（最简单，推荐）**
```bash
./start.bat
```

**方法B：手动启动**
```bash
npm run dev
```

成功后会看到：
```
VITE v5.x.x  ready in xxx ms

➜  Local:   http://localhost:5173/LearningOS/
```

---

#### 4. 访问应用

在浏览器中打开：
- 📊 **看板页**：http://localhost:5173/LearningOS/app/index.html
- 🎤 **录音页**：http://localhost:5173/LearningOS/app/recorder.html
- 📁 **文件夹**：http://localhost:5173/LearningOS/app/folder.html?date=2026-04-20

---

### 🐛 故障排除清单

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| `npm: command not found` | Node.js 未安装或未重启终端 | 安装 Node.js 并重启终端 |
| `package.json not found` | 目录错误 | `cd LearningOS` 进入子目录 |
| 录音按钮无反应 | 使用了 file:// 协议 | 必须通过 localhost:5173 访问 |
| 麦克风权限被拒绝 | 浏览器安全策略 | 点击地址栏图标允许权限 |
| Console 报错 "getUserMedia" | 环境不符合要求 | 确保使用 localhost 而非 file:// |

## 🎤 如何录音

### 打开录音页面

点击看板页上的 **"🎤 去录音"** 按钮，进入 `/app/recorder.html`

### 开始录音

1. 点击 **"▶ 开始录音"** 按钮
2. 系统会请求麦克风权限（首次使用）
3. 授予权限后，页面顶部会显示 **"🔴 正在录音..."** 倒计时

### 停止录音

1. 点击 **"■ 停止录音"** 按钮
2. 系统会自动保存音频到本地存储

**保存内容包括**:
- 音频 Blob 文件
- 当前日期文件夹信息
- 复习计划（[0,1,3,7] 天）

---

## 📻 播放和复习

### 在看板页播放音频

1. 找到对应日期文件夹下的音频
2. 点击音频旁的播放器 ▶
3. 可以调整音量和进度

### 标记为完成

1. 播放完音频后，点击 **"✓ 标记完成"** 按钮
2. 该项目会从看板中消失
3. 系统自动更新本地存储

## 📁 文件夹系统

### 自动分组

所有录音会按**创建日期**自动分组：

```
📁 2026-04-20
  - 音频1
  - 音频2
  
📁 2026-04-19
  - 音频3
```

### 复习显示逻辑

看板页**只显示**满足两个条件的音频：

1. **创建日期的复习计划到期了**（根据艾宾浩斯算法）
2. **尚未标记为完成** (done = false)

---

## 🧠 艾宾浩斯算法

每条新录音会自动生成 4 个复习副本：

| 复习次数 | 时间 | 说明 |
|---------|------|------|
| 第1次 | 今天 | 立即复习（巩固印象） |
| 第2次 | 1天后 | 强化记忆 |
| 第3次 | 3天后 | 加深理解 |
| 第4次 | 7天后 | 长期记忆 |

**示例**:
- 2026-04-20 录制的音频
- 会在以下日期出现在看板中：
  - ✓ 2026-04-20（当天）
  - ✓ 2026-04-21（1天后）
  - ✓ 2026-04-23（3天后）
  - ✓ 2026-04-27（7天后）

## 💾 数据存储

### 存储位置

所有数据保存在浏览器的 **IndexedDB** 中（不在服务器）

### 数据持久化

- ✅ 关闭浏览器：数据保留
- ✅ 刷新页面：数据保留
- ✅ 重新打开浏览器：数据保留
- ✅ 清空浏览历史（谨慎操作）：数据可能丢失

### 备份建议

暂无官方备份功能（计划在第三阶段实现）

**临时方案**：
- 定期手动下载音频
- 记录重要的学习数据

## 🐛 常见问题

### Q1: 为什么看不到之前录音的音频？

**可能原因**:
1. 该音频的复习计划还未到达今天
2. 该音频已被标记为完成

**解决方案**:
- 等待复习计划到期（根据艾宾浩斯算法）
- 或创建新的录音进行复习

---

### Q2: 浏览器提示"需要麦克风权限"

**解决方案**:
1. 点击 "▶ 开始录音" 时，浏览器会弹出权限请求
2. 选择 **"允许"**
3. 重新点击 "▶ 开始录音" 即可

**如果还是不行**:
- 检查系统是否允许浏览器访问麦克风
- 尝试用无痕模式打开页面
- 清除浏览器缓存后重试

---

### Q3: 音频文件被删除了怎么办？

**暂无恢复机制**（第一阶段限制）

**防护建议**:
1. 定期备份重要音频
2. 避免清空浏览历史
3. 不要卸载浏览器配置数据

---

### Q4: 如何导出我的学习数据？

第一阶段不支持导出（计划在第三阶段实现）

**临时方案**:
- 使用浏览器开发者工具（F12）
- 访问 Application → IndexedDB → learning_os → audios
- 可以查看所有存储的数据
